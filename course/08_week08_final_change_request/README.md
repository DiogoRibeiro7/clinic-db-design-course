# Week 08: Final change request

## Learning goals

- Implement a schema change safely.
- Write migration steps and validation queries.
- Communicate assumptions and design decisions.

## Run order

1) Choose one change request (A, B, or C)
2) Update ER diagram and schema
3) Fill migration_template.sql
4) Run validation queries

## Deliverables

- Selected change_request_*.md notes
- migration_template.sql (completed)
- final_submission_checklist.md (completed)

## Grading criteria

- Migration is complete and safe
- Validation queries return 0 rows
- Assumptions and decisions documented
- Schema changes are coherent
